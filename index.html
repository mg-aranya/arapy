Spec-Driven Network CLI

netloom

A modular network API CLI for operators and automation engineers. `netloom` discovers vendor APIs, keeps the command surface consistent, and now supports access-aware ClearPass catalog visibility so users only see what their API client can actually use.

View on GitHub View on PyPI Project Home 
$ pip install netloom-tool
$ netloom load clearpass
$ netloom server use discovery
$ netloom cache update
$ netloom ?

Visible modules:
  - apioperations
  - identities

$ netloom --catalog-view=full ?

Full vendor catalog:
  - apioperations
  - globalserverconfiguration
  - policyelements
  - ...

What It Does

`netloom` turns vendor API documentation into a usable CLI with discovery, help, filtering, copying, and profile-aware workflows.

  • Dynamic module, service, and action discovery
  • ClearPass-first plugin model
  • Context-aware help and shell completion
  • JSON, CSV, and raw output support

Why v1.8.2 Matters

The ClearPass cache now keeps two views at once: a strict default visible catalog for day-to-day use and an explicit full catalog view for troubleshooting and vendor-doc comparisons.

netloom --catalog-view=full ?
